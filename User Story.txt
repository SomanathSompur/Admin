***************** Cart MS *****************

User Story 1 : Add Item to Cart

Description:
As a user, I want to add items to my shopping cart, so I can review and proceed to purchase them.

Acceptance Criteria:

1.The user should be able to add the product to the cart
  (Each product should have an "Add to Cart" button.)
2.Clicking the "Add to Cart" button should update the contents of the user's shopping cart.
  (Clicking the button adds the item to my cart with the default quantity of 1.)
3.The shopping cart should display the added items, including their names, quantities, and prices.
  (If the item is already in the cart, the quantity should increment by 1.)

************************************************************************

User Story 2 : View and Modify Cart Contents

Description:
As a user, I want to view the contents of my shopping cart and have the ability to modify the 
quantity or remove items.

Acceptance Criteria:

1.There should be a "View Cart" option accessible from the user interface.
  (There should be a "View Cart" button accessible from any page.)
2.The "View Cart" page should display a list of items in the cart, including names, quantities, 
  prices, and a total.
  (In the cart view, I see a list of items with their names, quantities, and prices.)
3.Users should have the option to increase, decrease, or remove items from the cart.
  (For each item, I can easily adjust the quantity or remove it from the cart.)
4.The total price should update dynamically based on the changes made.

************************************************************************

User Story 3 : Checkout Process

Description:
As a user, I want a straightforward checkout process to complete my purchase efficiently.

Acceptance Criteria:

1.There should be a "Checkout" button on the "View Cart" page.
  (There should be a prominent "Checkout" button in the cart view.)
2.Clicking the "Checkout" button should redirect the user to a checkout page.
  (During checkout, I enter or confirm my delivery address.)
3.The checkout page should display a summary of the order, including items, quantities, 
  and the total price.
  (I can review the order summary, including item details and total price.)
4.Users should be prompted to enter shipping and payment information.
  (I provide payment details and complete the purchase with a single click.)
5.After completing the checkout process, the user should receive a confirmation of the order.
